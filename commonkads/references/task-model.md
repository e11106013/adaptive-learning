# CommonKADS 分析 Skill

## 概述
CommonKADS 是系統化的知識系統開發方法論。本 skill 引導完整的六模型分析。

## 何時讀取參考文件
- 用戶要求分析特定模型時 → 讀取對應的 references/*.md
- 需要填寫工作表時 → 讀取 references/templates/worksheet.md
- 快速概覽時 → 只用 SKILL.md 本身即可

## 分析流程

### 第一步：釐清分析範疇
詢問用戶：
1. 目標組織/系統是什麼？
2. 要解決的核心問題？
3. 分析深度（概覽/完整分析）？

### 第二步：組織模型 (OM-1 ~ OM-6)
分析六張工作表：
- OM-1：組織情境
- OM-2：問題與機會描述
- OM-3：處理變體描述
- OM-4：資產描述
- OM-5：知識系統行動計畫
- OM-6：可行性評估

→ 詳細指引見 references/organization-model.md

### 第三步：任務模型 (TM-1 ~ TM-2)
- TM-1：任務描述
- TM-2：知識項目描述

### 第四步：代理模型 (AM-1)
- AM-1：代理描述（角色、能力、權限）

### 第五步：知識模型 (KM)
最核心的模型，分析：
- 領域知識結構
- 推理知識
- 任務知識

→ 詳細指引見 references/knowledge-model.md

### 第六步：推理模型 (RM)
- 推理步驟分解
- 知識角色對應

### 輸出格式
每個模型輸出一份結構化文件，包含：
- 工作表編號與標題
- 填寫內容
- 發現的知識差距
- 建議行動

## 快速模式
如用戶只需要快速評估，可只執行 OM-1、OM-2、OM-5 三張工作表。
